# Plan 06-03: Admin Dashboard Tablet Support

<objective>
Ensure all admin dashboard pages are readable and functional on a tablet (>=768px). Tables must scroll horizontally rather than overflowing or wrapping into unreadable layouts. Dashboard KPI cards and charts must adapt to narrower widths (RES-06).
</objective>

<must_haves>
## Goal-Backward Verification

- truths:
  - "Admin dashboard tables on a tablet (>=768px) scroll horizontally rather than overflowing"
  - "Admin product edit modal is fully usable on a 768px screen"
  - "Admin dashboard KPI cards stack properly on tablet width"
</must_haves>

## Tasks

<task id="03-01" type="execute">
<title>Admin products table: horizontal scroll and column management</title>

<read_first>
- frontend/src/app/admin/products/page.tsx (table layout lines 218-278, modal lines 280-537)
</read_first>

<action>
In `admin/products/page.tsx`:

1. **Table wrapper** (line 218): Already has `overflow-x-auto` — verify it works. Add `min-width` to the table to force horizontal scroll on narrow widths:
```tsx
<div className="bg-white rounded-xl shadow-sm overflow-x-auto">
  <table className="w-full text-sm" style={{ minWidth: 700 }}>
```
This ensures the table maintains its structure and scrolls horizontally when the viewport is less than 700px + sidebar width.

2. **Hide less-critical columns on small tablets**: Add responsive visibility to the "Kích hoạt" (Active) and "Danh mục" (Category) columns on narrow admin views:
- Add `className="hidden lg:table-cell"` to the "Danh mục" th and corresponding td elements
- Keep "Ảnh", "Tên sản phẩm", "Giá", "Tồn kho", and "Thao tác" always visible

Specifically:
- Line 224 (`<th>Danh mục</th>`): Add `className="text-left px-4 py-3 hidden lg:table-cell"`
- Line 251 (category `<td>`): Add `className="px-4 py-3 text-gray-500 hidden lg:table-cell"`

3. **Product name truncation** (line 250): Ensure `max-w-[200px] truncate` works with `min-width` on table:
```tsx
<td className="px-4 py-3 font-medium max-w-[160px] lg:max-w-[200px] truncate">{p.name}</td>
```

4. **Search + Add button row** (line 211): Already uses `flex flex-col sm:flex-row` — verify.

5. **Modal** (line 282): Change `max-w-3xl` to be width-constrained on tablets:
```tsx
<div className="bg-white rounded-2xl w-full max-w-[95vw] md:max-w-3xl max-h-[90vh] overflow-y-auto shadow-2xl">
```
This ensures the modal doesn't overflow on 768px screens.

6. **Modal form grid** (line 290): Already uses `grid-cols-1 md:grid-cols-2` — verify.
</action>

<acceptance_criteria>
- grep "minWidth: 700" frontend/src/app/admin/products/page.tsx returns a match
- grep "hidden lg:table-cell" frontend/src/app/admin/products/page.tsx returns 2 matches (th + td for category column)
- grep 'max-w-\[95vw\] md:max-w-3xl' frontend/src/app/admin/products/page.tsx returns a match
- At 768px with sidebar (effective ~512px content), products table scrolls horizontally
- Product edit modal is fully visible and scrollable at 768px
</acceptance_criteria>
</task>
